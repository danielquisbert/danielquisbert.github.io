Diagrama imagen vs contenedor
Imagen
Plantilla de solo lectura. Como un instalador congelado. Se descarga una vez.
Contenedor
Instancia en ejecución de una imagen. Puede haber varios del mismo tipo.
Volumen
Directorio del host montado en el contenedor. Los datos persisten aunque el contenedor se elimine.
GeoServer 3 en un comando
docker run -d \ --name geoserver3 \ -p 8080:8080 \ -v /opt/geoserver_data:/opt/geoserver/data_dir \ -e EXTRA_JAVA_OPTS="-Xms512m -Xmx2g" \ docker.osgeo.org/geoserver:3.0.0 # Ver logs en vivo docker logs -f geoserver3 # Estado docker ps
Docker Compose
GeoServer + PostGIS — el stack completo
docker-compose.yml
services: postgis: image: postgis/postgis:16-3.4 environment: POSTGRES_DB: geobolivia POSTGRES_USER: geouser POSTGRES_PASSWORD: Bolivia2026! volumes: - pgdata:/var/lib/postgresql/data geoserver: image: docker.osgeo.org/geoserver:3.0.0 ports: ["8080:8080"] environment: GEOSERVER_DATA_DIR: /opt/geoserver/data_dir volumes: - gsdata:/opt/geoserver/data_dir depends_on: [postgis] volumes: pgdata: {} gsdata: {}
▶
docker compose up -d — levanta todo el stack. docker compose down — lo detiene.

Formatos vectoriales
Shapefile vs GeoPackage — la migración necesaria
⚠️
Limitaciones del Shapefile
Nombres de campo máx. 10 caracteres · Máximo 2 GB por archivo · Requiere 3+ archivos separados · Sin soporte para fechas/horas completas
✓
GeoPackage lo resuelve
Un único archivo .gpkg · Nombres de campo hasta 255 chars · Sin límite de tamaño práctico · Múltiples capas en un archivo · Basado en SQLite estándar

Sección 3 de 3
ogr2ogr
la navaja suiza
Conversión, reproyección y filtrado de datos geoespaciales
ogr2ogr esencial
Conversiones más usadas en el curso
Shapefile → GeoPackage con reproyección
# SHP a GeoPackage en WGS84 ogr2ogr -f GPKG municipios.gpkg municipios.shp \ -t_srs EPSG:4326 -nln municipios # Verificar el resultado ogrinfo municipios.gpkg -al -so
GeoPackage → PostGIS (importar a la base de datos)
ogr2ogr -f PostgreSQL \ PG:"host=localhost dbname=geobolivia user=geouser" \ municipios.gpkg -nln municipios \ -t_srs EPSG:4326 -overwrite
GeoPackage → GeoJSON
ogr2ogr -f GeoJSON departamentos.geojson \ departamentos.gpkg -t_srs EPSG:4326
Actividad práctica
Ejercicio: del SHP al servidor en 4 pasos
1
Descarga los datos del ejercicio
Shapefiles de departamentos y municipios provistos por el instructor.

Convierte a GeoPackage con ogr2ogr
ogr2ogr -f GPKG datos.gpkg departamentos.shp -t_srs EPSG:4326
3
Levanta el stack con Docker Compose
docker compose up -d — verifica que GeoServer y PostGIS arrancan.
4
Copia el GeoPackage al Data Directory
Y verifica que GeoServer puede leerlo desde la interfaz web.
Resumen
Lo que aprendiste hoy
  • Docker usa imágenes (plantillas) para crear contenedores (instancias en ejecución)
  • Los volúmenes persisten los datos fuera del contenedor
  • docker-compose.yml orquesta GeoServer + PostGIS con un solo comando
  • GeoPackage reemplaza al Shapefile: un archivo, sin límites de nombre, sin límite de 2GB
  • ogr2ogr convierte, reproyecta y filtra datos en un solo comando
