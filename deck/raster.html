Flujo de publicación
Las pestañas clave al publicar una capa
📋
Pestaña Data
Nombre: identificador técnico (en URL)
Título: texto legible para usuarios
CRS Nativo: verificar EPSG
Bounding Box: usar "Compute from data"
Keywords: español + inglés
🎨
Pestaña Publishing
Estilo por defecto: seleccionar o crear
WMS habilitado: sí
WFS habilitado: según necesidad
Queryable: para GetFeatureInfo
Opacidad: 1.0 por defecto
⚠️
El error más común
Publicar sin hacer clic en "Compute from data" para el Bounding Box. La capa aparece pero no se renderiza correctamente en el preview.

Publicar capas
raster
GeoTIFF y Modelo Digital de Elevación
Datos raster
Publicar un DEM (Modelo Digital de Elevación)
1
Store tipo "GeoTIFF"
Apunta al archivo .tif del DEM en el Data Directory.
2
Verificar CRS y Bounding Box
DEM en EPSG:4326, extensión completa del área de estudio.
3
Configurar NoData = -9999
Valores sin dato aparecerán transparentes en el WMS.
4
Asignar estilo "dem" por defecto
GeoServer incluye un estilo DEM con gradiente de colores.

Paso 4 de 4
Layer Groups
Múltiples capas, un solo nombre en el WMS
Composición de mapas
Layer Group — el mapa base completo
💡
¿Para qué sirve?
Un cliente pide LAYERS=mapa_base y GeoServer renderiza: DEM + hidrografía + vías + departamentos + ciudades. Todo en una sola imagen.
1
Datos → Layer Groups → Nuevo
Nombre: mapa_base
2
Agregar capas en orden
Abajo = fondo (DEM), Arriba = encima (ciudades).
3
Generar Bounding Box
"Generate Bounds" calcula la unión de todas las capas.

Resumen
Lo que aprendiste hoy
  • El flujo siempre es: Workspace → Store → Publicar Layer
  • Nombre técnico e invisible vs Título legible para el usuario — son campos distintos
  • "Compute from data" es obligatorio para que el Bounding Box sea correcto
  • Las capas raster necesitan configurar el valor NoData para mostrar transparencia
  • Un Layer Group combina varias capas en un mapa servido con un solo nombre
